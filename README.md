# RelayWatch

RelayWatch is an AI relay discovery and monitoring dashboard. It aggregates
relay sites, compares model pricing, tracks announcements, shows official API
status, collects AI news/tutorials, and provides lightweight chat and model
detection tools.

## Features

- Site aggregation with health status, model count, lowest ratio, notices, and
  detail drawers.
- Model price comparison across sites, groups, billing modes, and performance
  samples.
- Announcement feed with search, tags, deduplication-friendly timestamps, and
  public-site notices.
- Official API status summaries for upstream providers.
- AI news, tutorials, community discussions, and open-source project feeds.
- Online chat page for OpenAI-compatible endpoints.
- Model/site detection page with protocol and quality reports.
- FastAPI backend, React/Vite frontend, JSON fallback mode, and optional
  PostgreSQL storage.

## Repository Layout

```text
relaywatch/
  server.py              FastAPI app and API endpoints
  normalize_data.py      Convert collected raw results into JSON data
  load_json_to_db.py     Import normalized JSON into PostgreSQL
  incremental_import.py  Incremental PostgreSQL import helpers
  db_schema.sql          PostgreSQL schema
  web/                   React + Vite frontend source
  static/                Runtime static directory generated by frontend build
  data/                  Generated local JSON data, ignored by git
```

## Quick Start

Install Python dependencies:

```bash
cd relaywatch
python -m pip install -r requirements.txt
```

Install and build the frontend:

```bash
cd web
npm install
npm run build
cd ..
```

Generate local JSON data from a raw collection file:

```bash
python normalize_data.py --input ../api_config_results.json --out-dir data
```

Start the app:

```bash
python -m uvicorn server:app --host 127.0.0.1 --port 8765
```

Open `http://127.0.0.1:8765`.

## Configuration

Copy `.env.example` to `.env` for local deployment and fill in your own values.
Never commit real tokens, database passwords, cookies, or API keys.

Important variables:

- `DATABASE_URL`: enables PostgreSQL mode when set.
- `RELAYWATCH_ADMIN_TOKEN`: protects admin-only endpoints.
- `RELAYWATCH_DETECTOR_BASE_URL`: optional detector service for model checks.
- `GITHUB_TOKEN` or `RELAYWATCH_GITHUB_TOKEN`: optional GitHub API token for
  higher collection rate limits.
- `RELAYWATCH_LINUXDO_COOKIE`: optional cookie for feeds protected by
  Cloudflare.

## Data Storage

RelayWatch supports two modes:

- JSON mode: reads generated files from `data/sites.json`, `data/models.json`,
  `data/announcements.json`, and `data/summary.json`.
- PostgreSQL mode: set `DATABASE_URL`, import data with `load_json_to_db.py`,
  and the API reads from versioned database tables.

Example PostgreSQL import:

```bash
export DATABASE_URL='postgresql://relaywatch:change-me@127.0.0.1:5432/relaywatch'

python load_json_to_db.py \
  --data-dir data \
  --schema db_schema.sql \
  --kind json_import
```

The importer writes a new data generation and then atomically switches the
active generation. If an import fails, the previous generation remains active.

## Open Source Notes

Generated data, runtime logs, local caches, frontend build output, and secrets
are ignored by git. Public repositories should contain source code and example
configuration only.

Before publishing, run a secret scan for API keys, passwords, cookies, private
URLs, and database dumps.
