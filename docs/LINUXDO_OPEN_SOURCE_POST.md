## 项目介绍

大家好，最近折腾了一个小项目，叫 **RelayWatch**。

它主要是面向 NewAPI / Sub2API / AI API 中转站生态的聚合监控工具，想解决一个比较朴素的问题：

> 现在各种中转站、模型分组、倍率、公告、维护通知太分散了，想找一个模型到底哪个站有、价格怎么样、接口还活不活，往往要一个个点进去看。

所以我做了一个可以自动采集、探活、归一化、比价和展示的看板，把公开可访问的中转站信息整理成一个可以搜索、筛选和对比的目录。

## 现在能做什么

### 1. 站点聚合

自动整理公开互联网上可访问的 NewAPI / Sub2API 类站点，展示站点状态、模型数量、最低倍率、公告、可用分组等信息。

### 2. 模型比价

可以按模型维度查看不同站点的价格、倍率、输入价格、输出价格、缓存价格、成功率、延迟和 TPS。

比如想看某个 GPT、Claude、Gemini、DeepSeek 模型在哪些站点有，就不用一个个网站翻了。

### 3. 公告流

聚合各个站点的公告、维护通知、价格调整、活动优惠等内容，并保留时间线。

这个功能主要是为了看哪些站点最近在维护、涨价、迁移域名或者调整模型。

### 4. 官方 API 状态

聚合 OpenAI、Claude、Gemini、DeepSeek 等官方状态页，方便判断是上游波动，还是中转站自己的问题。

### 5. 模型检测

可以输入自己的 API Base 和 Key，获取模型列表，然后对指定模型做协议检测、调用检测和质量检测。

这个功能还在继续优化，目标是让大家能更直观地判断某个站点的模型是不是可用、响应是否正常、质量大概怎么样。

### 6. 在线对话

内置了一个简单的 OpenAI 兼容接口聊天页面，支持自动拉取模型列表和流式输出，方便直接试接口。

### 7. AI 资讯和教程聚合

也做了一个 AI 热点资讯、教程文章、社区讨论的聚合页，目前还在调整数据源和展示方式。

## 为什么做这个

主要是自己用的时候感觉太麻烦了。

现在 AI API 中转站很多，但信息非常碎：

- 有的站点公告写在前台
- 有的模型价格藏在接口里
- 有的站点能打开但模型列表是空的
- 有的模型名字一样但倍率差很多
- 有些公告没有时间
- 有些站点临时维护、换域名，不看公告根本不知道

所以就想做一个比较中立的聚合工具，把这些公开信息自动整理出来。

它不是推荐某个站，也不是保证某个站一定可靠，只是尽量把公开数据结构化，让大家少一点重复查找的时间。

## 部署方式

项目支持两种运行方式：

1. 本地 JSON 文件模式，适合自己跑着玩
2. PostgreSQL 模式，适合长期线上运行和大数据量检索

简单启动大概是：

```bash
cd relaywatch
python -m pip install -r requirements.txt

cd web
npm install
npm run build
cd ..

python -m uvicorn server:app --host 127.0.0.1 --port 8765
```

具体环境变量、数据库导入和采集脚本可以看 README。
